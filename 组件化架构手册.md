## 组件分层原则

### 组件大小定义

```
┌─────────────────────────────────────────┐
│          页面框架 (整体页面结构)          │
│  ┌─────────────────────────────────────┐ │
│  │       大组件 (页面主要功能区)         │ │
│  │  ┌────────────────────────────────┐ │ │
│  │  │     中组件 (功能区内的模块)      │ │ │
│  │  │  ┌──────────────────────────┐  │ │ │
│  │  │  │  小组件 (模块内的元素)    │  │ │ │
│  │  │  │  ┌────────────────────┐  │  │ │ │
│  │  │  │  │ 最小组件 (基础元素) │  │  │ │ │
│  │  │  │  │ • 图标 + 文字      │  │  │ │ │
│  │  │  │  │ • 按钮             │  │  │ │ │
│  │  │  │  │ • 文字             │  │  │ │ │
│  │  │  │  │ • 图片             │  │  │ │ │
│  │  │  │  └────────────────────┘  │  │ │ │
│  │  │  └──────────────────────────┘  │  │ │
│  │  └────────────────────────────────┘ │ │
│  └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

### 组件分类说明

| 层级 | 名称 | 定义 | 示例 |
|------|------|------|------|
| **页面框架** | Page | 整个页面的布局结构 | 首页、购买页 |
| **大组件** | Large | 页面内的主要功能区块 | 顶部导航、轮播图、筛选栏 |
| **中组件** | Medium | 大组件内的功能模块 | 搜索框、行业标签、服务卡片 |
| **小组件** | Small | 中组件内的界面元素 | 标签、排序项、价格标签 |
| **最小组件** | Atom | 不可再分的界面元素 | 图标、文字、按钮、图片 |

```

│   │   ├── 背景图片或视频 (最小组件)
│   │   ├── 内容容器 (中组件)
│   │   │   ├── 标题文字 (最小组件)

- 鼠标悬停时暂停轮播
- 可以点击圆点手动切换

**展示内容**:
- 第一屏：宣传视频（自动播放）
- 第二屏：宣传图片

- 支持标题、列表、段落

**5. 案例展示**
- 展示完整的案例页面
- 例如：智慧景区导览系统
- 全屏展示，无顶部栏和标签

4. "服务详情页"重新构建页面
5. "侧边操作按钮"收到新的状态，将按钮文字从"关注"改为"已关注"，图标变为实心红色
6. 页面底部显示提示消息："已关注"

```
首页 (父组件)

#### 通信流程说明

**流程1：轮播图自动切换**

1. "首页轮播大图"组件的定时器触发：5秒时间到
2. 组件更新内部状态：将【当前轮播索引】从0改为1
3. "轮播滑动器"收到新的状态，从第1张图片滑动到第2张图片
4. "轮播指示器"收到新的状态，将第2个圆点高亮显示

**流程2：用户点击轮播指示器**

1. 用户在"轮播指示器"中点击第3个圆点
2. "轮播指示器"向"首页轮播大图"发送通知：用户选中了第3张
3. "首页轮播大图"收到通知后更新内部状态：将【当前轮播索引】改为2
4. "轮播滑动器"收到新的状态，滑动到第3张图片
5. "轮播指示器"收到新的状态，高亮显示第3个圆点

## 性能优化说明

### 1. 图片优化

**网络图片加载**:
- 使用缓存机制，避免重复加载
- 加载失败时显示占位图
- 加载中显示加载指示器

**图片尺寸**:
- 根据显示尺寸请求合适大小的图片
- 避免加载过大的图片浪费流量

---

### 2. 动画优化

**动画控制**:
- 使用动画控制器，避免创建不必要的动画
- 动画结束后及时释放资源

**页面切换**:
- 页面切换时暂停视频播放
- 页面切换时停止轮播图自动播放

---

### 3. 列表优化

**网格布局**:
- 使用虚拟滚动，只渲染可见区域的卡片
- 滚动时动态加载更多内容

**筛选优化**:
- 先筛选数据，再渲染界面
- 避免不必要的界面重建
