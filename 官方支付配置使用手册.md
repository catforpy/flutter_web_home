# 支付配置页面使用手册

## 📋 版本说明

本项目包含**三个版本**的支付配置页面，分别针对不同的使用场景：

### 版本对比

| 版本 | 文件名 | 风格 | 目标用户 | 适用场景 |
|------|--------|------|----------|----------|
| **旧版表单** | `payment_config_content.dart` | 传统表单 | 技术人员 | 手动配置已有商户号 |
| **PC版工作台** | `payment_config_v2.dart` | 企业级SaaS | 商户运营 | 批量管理多个进件 |
| **移动端风格** | `mobile_style_payment_config.dart` | 微信小程序风格 | 中小商户老板 | 单次进件申请 |

## 📱 移动端风格版本详细指南

### 核心特性

#### 1. 4步填报向导

```
步骤 1/4：主体资质
  ├─ 商户简称
  ├─ 主体类型（企业/个体户/其他）
  ├─ 营业执照上传（支持OCR自动识别）
  ├─ 统一社会信用代码
  ├─ 执照名称
  ├─ 法人姓名
  └─ 有效期

步骤 2/4：管理员信息
  ├─ 管理员类型（法人本人/代理人）
  ├─ 姓名
  ├─ 身份证号（掩码显示）
  ├─ 身份证照片（正反面）
  ├─ 手机号（短信验证）
  └─ 微信号（可选）

步骤 3/4：结算账户
  ├─ 账户类型（对公账户/个人卡）
  ├─ 开户名称
  ├─ 银行账号（掩码显示）
  └─ 开户银行（级联选择）

步骤 4/4：经营场景
  ├─ 经营场景勾选（多选）
  │   ├─ 线下门店 → 动态显示门店信息
  │   ├─ 微信小程序 → 动态显示AppID
  │   └─ 公众号/APP/网站
  └─ 客服电话
```

#### 2. 交互亮点

##### OCR智能识别
- 上传营业执照后自动识别
- 自动填充：信用代码、执照名称、法人姓名
- 绿色高亮显示供核对
- 识别过程显示Loading动画

##### 掩码显示
- 身份证号：`3301**********1234`
- 银行账号：`6222 **** **** 1234`
- 点击"👁️"图标可查看明文

##### 表单验证
- 实时校验输入格式
- 错误字段红色高亮
- 自动滚动到第一个错误位置

##### 动态表单
- 勾选"线下门店"：显示门店名称、地址、门头照上传
- 勾选"小程序"：显示AppID、截图上传
- 未勾选的字段隐藏，保持界面简洁

#### 3. 进度看板页面

提交后自动跳转到 `mobile_style_progress_page.dart`

**状态类型**:

| 状态 | 背景色 | 显示内容 |
|------|--------|----------|
| 🟡 审核中 | 淡黄色 #FFFBE6 | 时钟图标 + 预计时间 |
| 🔵 待验证/待签约 | 淡蓝色 #E6F7FF | 二维码 + 倒计时 |
| 🟢 已完成 | 淡绿色 #F6FFED | 对勾 + 商户号信息 |
| 🔴 已驳回 | 淡红色 #FFF1F0 | 驳回原因 + 修改按钮 |

**核心功能**:
- **二维码区**: 实时显示微信扫码授权码，5分钟倒计时
- **进度时间轴**: 5个节点，清晰展示当前进度
- **状态自动刷新**: 模拟轮询（实际项目中对接后端WebSocket）
- **客服联系**: 一键联系客服，显示客服电话

## 🔄 版本切换

### 切换到旧版表单

**适用场景**: 已有商户号，只需配置API密钥等技术参数

**方法**:
1. 打开 `lib/presentation/pages/workbench/mini_program_management_page.dart`
2. 找到第279行的支付配置判断
3. 将 `MobileStylePaymentConfig()` 改为 `PaymentConfigContent()`

```dart
// 当前代码（移动端风格）
if (subItem == '支付配置') {
  Navigator.push(
    context,
    MaterialPageRoute(
      builder: (context) => const MobileStylePaymentConfig(),  // ← 移动端风格
    ),
  );
}

// 修改为（旧版表单）
if (subItem == '支付配置') {
  Navigator.push(
    context,
    MaterialPageRoute(
      builder: (context) => const PaymentConfigContent(),  // ← 旧版表单
    ),
  );
}
```

### 切换到PC版工作台

**适用场景**: 需要批量管理多个进件，查看统计Dashboard

**方法**:
```dart
if (subItem == '支付配置') {
  Navigator.push(
    context,
    MaterialPageRoute(
      builder: (context) => const PaymentConfigV2Page(),  // ← PC版工作台
    ),
  );
}
```

## 📊 数据流

### 提交流程

```
用户填写表单
    ↓
4步数据暂存（内存）
    ↓
点击"提交申请"
    ↓
显示Loading遮罩
    ↓
调用后端API（实际项目中）
    ↓
返回成功
    ↓
跳转到进度看板页面
    ↓
开始轮询状态（每5秒）
```

### 状态更新

```dart
// 模拟轮询（实际项目中使用WebSocket）
Timer.periodic(Duration(seconds: 5), (timer) {
  _fetchStatus();  // 请求后端接口
  if (statusChanged) {
    setState(() {
      _currentStatus = newStatus;  // 更新UI
    });
  }
});
```

## 💡 最佳实践

### 1. 表单设计

- ✅ **字段分组**: 相关字段放在一起，用标签和提示分隔
- ✅ **即时验证**: 用户输入后立即校验，不要等到提交
- ✅ **友好提示**: 占位符、帮助文字、示例格式
- ✅ **掩码保护**: 敏感字段默认掩码，提供查看按钮

### 2. 交互反馈

- ✅ **上传进度**: 显示上传进度百分比
- ✅ **Loading状态**: 提交时全屏遮罩，防止重复点击
- ✅ **成功提示**: 使用SnackBar显示成功消息
- ✅ **错误提示**: 错误字段红色边框 + 错误文案

### 3. 移动端适配

虽然是PC Web，但借鉴移动端设计：

- ✅ **大按钮**: 最小44px高度，方便点击
- ✅ **单列布局**: 避免横向滚动
- ✅ **固定导航栏**: 顶部和底部固定，中间滚动
- ✅ **安全区域**: 底部留出padding，避免内容被遮挡

## 🔧 常见问题

### Q1: 如何添加新的经营场景？

A: 在 `_buildStep4BusinessScenes()` 中修改scenes数组：

```dart
final scenes = ['线下门店', '微信小程序', '公众号', 'APP', '网站', 'H5页面'];
```

然后在 `_buildDynamicFormFields()` 中添加对应的表单字段。

### Q2: 如何修改主题色？

A: 修改文件顶部的颜色常量：

```dart
const Color wxGreen = Color(0xFF07C160);  // 改为你的品牌色
```

### Q3: 如何对接真实的后端API？

A: 找到 `_submitApplication()` 方法，替换模拟代码：

```dart
// 原代码（模拟）
Future.delayed(const Duration(seconds: 2), () {
  Navigator.pop(context);
  Navigator.pushReplacement(...);
});

// 修改为（真实API）
final response = await http.post(
  Uri.parse('https://your-api.com/merchant/apply'),
  body: json.encode(_formData),
);
if (response.statusCode == 200) {
  Navigator.pushReplacement(...);
}
```

### Q4: 二维码如何从后端获取？

A: 在 `_buildQrCodeSection()` 中，修改 `QrImageView` 的data参数：

```dart
// 原代码（模拟）
QrImageView(
  data: 'https://open.weixin.qq.com/...',  // 固定URL
)

// 修改为（真实API）
QrImageView(
  data: _authUrlFromApi,  // 从API获取的URL
)
```
