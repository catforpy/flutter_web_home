## 🚀 快速部署（推荐）

### 一键启动脚本

创建启动脚本 `start.sh`：

```bash
#!/bin/bash

# 进入项目目录
cd "/Volumes/DudaDate/微信小程序开发/flutter官网开发/official_website"

echo "==================================="
echo "   Flutter Web 本地部署启动脚本"
echo "==================================="
echo ""

# 1. 清理旧的构建
echo "📦 步骤 1: 清理旧的构建文件..."
flutter clean

# 2. 构建 Web 生产版本
echo ""
echo "🔨 步骤 2: 构建 Web 生产版本..."
flutter build web --release

# 3. 检查构建是否成功
if [ ! -d "build/web" ]; then
    echo "❌ 构建失败！请检查错误信息。"
    exit 1
fi

echo "✅ 构建成功！"

# 4. 停止旧的 Python 服务器（如果有）
echo ""
echo "🛑 步骤 3: 停止旧的服务器..."
pkill -f "python.*http.server.*8080" 2>/dev/null
sleep 2

# 5. 启动 Python HTTP 服务器
echo ""
echo "🚀 步骤 4: 启动 HTTP 服务器..."
cd build/web
python3 -m http.server 8080 &
SERVER_PID=$!

# 等待服务器启动
sleep 3

# 6. 检查服务器是否启动成功
if lsof -i :8080 > /dev/null 2>&1; then
    echo "✅ 服务器启动成功！"
    echo ""
    echo "==================================="
    echo "   🎉 部署完成！"
    echo "==================================="
    echo ""
    echo "📍 本地访问地址："
    echo "   http://localhost:8080"
    echo "   http://127.0.0.1:8080"
    echo ""
    echo "📍 局域网访问地址："
    echo "   http://192.168.2.14:8080"
    echo "   （请将 192.168.2.14 替换为你的实际 IP 地址）"
    echo ""
    echo "💡 提示：按 Ctrl+C 停止服务器"
    echo ""
    echo "==================================="

    # 自动打开浏览器
    open http://localhost:8080

    # 等待用户输入
    echo "按任意键停止服务器..."
    read -n 1

    # 停止服务器
    echo ""
    echo "🛑 正在停止服务器..."
    kill $SERVER_PID 2>/dev/null
    pkill -f "python.*http.server.*8080" 2>/dev/null
    echo "✅ 服务器已停止"
else
    echo "❌ 服务器启动失败！"
    exit 1
fi
```

### 使用方法

```bash
# 1. 创建启动脚本
cd "/Volumes/DudaDate/微信小程序开发/flutter官网开发/official_website"
cat > start.sh
# 粘贴上面的脚本内容
# 按 Ctrl+D 保存

# 2. 添加执行权限
chmod +x start.sh

# 3. 运行脚本
./start.sh
```

## 📚 详细步骤

### 步骤 1: 清理旧的构建

```bash
cd "/Volumes/DudaDate/微信小程序开发/flutter官网开发/official_website"

# 清理旧的构建文件
flutter clean
```

**作用**: 删除 `build/` 目录中的旧文件，确保从头开始构建。

---

### 步骤 2: 构建 Web 生产版本

```bash
flutter build web --release
```

**参数说明**：
- `build` - 构建应用
- `web` - 指定 Web 平台
- `--release` - 使用生产模式（优化性能）

**构建时间**：约 2-5 分钟

**成功标志**：
```
✓ Built build/web
```

**构建产物**：
```
build/web/
├── assets/
├── canvaskit/
├── index.html          ← 入口文件
├── main.dart.js        ← 编译后的 Dart 代码
├── flutter.js
└── ...
```

---

### 步骤 3: 停止旧的服务器

```bash
# 停止可能占用 8080 端口的进程
pkill -f "python.*http.server.*8080"
```

**验证端口已释放**：
```bash
lsof -i :8080
# 如果没有输出，说明端口已释放
```

---

### 步骤 4: 启动 HTTP 服务器

```bash
cd build/web
python3 -m http.server 8080
```

**参数说明**：
- `python3` - 使用 Python 3
- `-m http.server` - 启动 HTTP 服务器模块
- `8080` - 指定端口号

**成功标志**：
```
Serving HTTP on 0.0.0.0 port 8080 (http://0.0.0.0:8080/) ...
```

**后台运行**：
```bash
# 在后台运行服务器
python3 -m http.server 8080 &

# 查看进程
ps aux | grep "python.*http.server"
```

## 🔍 常见问题

### Q1: 端口被占用

**问题**：
```
OSError: [Errno 48] Address already in use
```

**解决**：
```bash
# 方法1: 停止占用端口的进程
lsof -i :8080
kill -9 <PID>

# 方法2: 使用其他端口
python3 -m http.server 8081
```

---

### Q2: 构建失败

**问题**：
```
Error: Could not resolve all packages.
```

**解决**：
```bash
# 清理并重新获取依赖
flutter clean
flutter pub get
flutter build web --release
```

---

### Q3: 页面显示空白

**可能原因**：
1. 浏览器缓存
2. 构建文件不完整

**解决**：
```bash
# 1. 清除浏览器缓存
# Chrome: Shift + Cmd + Delete

# 2. 重新构建
flutter clean
flutter build web --release

# 3. 重启服务器
pkill -f "python.*http.server.*8080"
# 然后重新启动
```

---

### Q4: 无法从局域网访问

**检查清单**：

1. **确认防火墙设置**
   ```bash
   # macOS 系统偏好设置 → 安全性与隐私 → 防火墙
   ```

2. **确认端口开放**
   ```bash
   # 检查服务器是否监听所有接口
   netstat -an | grep 8080
   # 应该看到 *.8080 或 0.0.0.0:8080
   ```

3. **确认在同一网络**
   - 确保访问设备和服务器在同一 Wi-Fi 或局域网

---

### Q5: 修改代码后看不到更新

**原因**: 使用的是生产构建版本，需要重新构建

**解决**：
```bash
# 1. 修改代码
# 2. 重新构建
flutter build web --release

# 3. 刷新浏览器
# Chrome: Cmd + R (硬刷新)
```

## 🛑 停止服务

### 方法 1: 使用进程 ID

```bash
# 1. 查找进程
ps aux | grep "python.*http.server"

# 2. 停止进程
kill <PID>
```

### 方法 2: 使用 pkill

```bash
pkill -f "python.*http.server.*8080"
```

如果服务器在前台运行，直接按 `Ctrl+C` 停止。

## 📊 部署对比

### 开发模式 vs 生产模式

| 特性 | 开发模式 (`flutter run`) | 生产模式 (`flutter build` + HTTP 服务器) |
|------|-------------------------|-------------------------------------|
| **启动方式** | `flutter run -d chrome` | `flutter build` + `python3 -m http.server` |
| **性能** | 较慢（包含调试信息） | 快速（优化编译） |
| **热重载** | ✅ 支持 | ❌ 不支持 |
| **调试** | ✅ 支持 | ❌ 不支持 |
| **访问方式** | 只能 localhost | 局域网可访问 |
| **稳定性** | 容易崩溃 | 稳定 |
| **适用场景** | 开发调试 | 演示、测试、生产 |

## 🎯 最佳实践

### 开发阶段
使用 `flutter run` 进行开发：
```bash
flutter run -d chrome --web-port 3000
```

**优点**：
- 热重载，快速迭代
- 可以调试代码
- 实时查看日志

### 测试/演示阶段
使用生产构建 + HTTP 服务器：
```bash
flutter build web --release
cd build/web
python3 -m http.server 8080
```

**优点**：
- 性能更好
- 可以从局域网访问
- 更加稳定
