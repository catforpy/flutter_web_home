# 课程问答管理系统设计方案

## 一、功能概述

实现一个课程问答管理系统，包含两个维度：
- **课前问答**：用户购买课程前向讲师/客服提出的问题
- **课后问答**：学习过程中由讲师回答的问题

## 二、课前问答 vs 课后问答 对比

### 2.1 核心差异

| 维度 | 课前问答 | 课后问答 |
|------|---------|---------|
| **主体** | 以课程为主体 | 以课程为主体 |
| **问题类型** | 购买前的咨询 | 学习过程中的问题 |
| **回答者** | 客服或讲师 | **只有讲师** |
| **典型问题** | "这个课程适合零基础吗？"<br/>"有优惠吗？"<br/>"包含资料吗？" | "这个知识点怎么理解？"<br/>"代码报错了怎么办？"<br/>"作业怎么做？" |

### 2.2 UI结构对比

**相同点**：
- ✅ 都以课程为主体展示
- ✅ 搜索筛选条件相同
- ✅ 课程列表布局相同
- ✅ 详情弹窗结构相同

**不同点**：
- ⚠️ 课前问答：回答者显示"📝 客服"或"👨‍🏫 讲师"
- ⚠️ 课后问答：回答者只显示"👨‍🏫 讲师"

## 三、数据结构设计

### 2.1 课程问答数据模型

```dart
/// 问答类型
enum QAType {
  preCourse('课前问答', 'pre_course'),
  postCourse('课后问答', 'post_course');

  final String label;
  final String value;
  const QAType(this.label, this.value);
}

/// 课程状态
enum CourseStatus {
  presale('预售', 'presale'),
  inProduction('制作中', 'in_production'),
  completed('完结', 'completed');

  final String label;
  final String value;
  const CourseStatus(this.label, this.value);
}

/// 回答人类型
enum AnswererType {
  customerService('客服', 'customer_service'),
  instructor('讲师', 'instructor');

  final String label;
  final String value;
  const AnswererType(this.label, this.value);
}

/// 单个问题
class Question {
  final String id;                  // 问题ID
  final String courseId;            // 课程ID
  final String courseName;          // 课程名称
  final String userId;              // 提问用户ID
  final String userName;            // 提问用户名
  final String userAvatar;          // 提问用户头像
  final String title;               // 问题标题
  final String content;             // 问题内容
  final DateTime askTime;           // 提问时间
  final bool isResolved;            // 是否已解决
  final AnswererType? answererType; // 回答人类型
  final String? answererName;       // 回答人姓名
  final DateTime? answerTime;       // 回答时间
  final String? answerContent;      // 回答内容
  final int likes;                  // 点赞数
  final QAType type;                // 问答类型

  const Question({
    required this.id,
    required this.courseId,
    required this.courseName,
    required this.userId,
    required this.userName,
    required this.userAvatar,
    required this.title,
    required this.content,
    required this.askTime,
    this.isResolved = false,
    this.answererType,
    this.answererName,
    this.answerTime,
    this.answerContent,
    this.likes = 0,
    required this.type,
  });
}

/// 课程问答统计
class CourseQAStats {
  final String courseId;            // 课程ID
  final String courseName;          // 课程名称
  final String instructorName;      // 讲师姓名
  final String instructorAvatar;    // 讲师头像

  // 分类信息
  final String? industryCategory;   // 行业分类（4层之一）
  final String? formatCategory;     // 课程形式分类
  final String? typeCategory;       // 课程类型分类

  final CourseStatus status;        // 课程状态
  final int followerCount;          // 关注人数
  final int studentCount;           // 学生人数
  final int questionCount;          // 问题数
  final int answerCount;            // 回答数
  final double rating;              // 评分
  final List<Question> questions;   // 问题列表

  const CourseQAStats({
    required this.courseId,
    required this.courseName,
    required this.instructorName,
    required this.instructorAvatar,
    this.industryCategory,
    this.formatCategory,
    this.typeCategory,
    required this.status,
    required this.followerCount,
    required this.studentCount,
    required this.questionCount,
    required this.answerCount,
    required this.rating,
    this.questions = const [],
  });
}
```

## 五、技术实现要点

### 5.1 筛选逻辑

#### 课前问答 & 课后问答（相同）：
1. **课程名称/讲师搜索**：模糊匹配课程名称或讲师姓名
2. **行业分类**：使用之前的三维分类系统
3. **课程形式**：预录/直播等
4. **课程类型**：单课/套课
5. **课程状态**：预售/制作中/完结

#### 详情页筛选（弹窗内）：
1. **解决状态**：全部/已解决/待解决
2. **时间排序**：最新/最早

### 5.2 UI组件

1. **TabBar**：课前问答 / 课后问答切换
2. **搜索框**：支持实时搜索
3. **DropdownButton**：多个筛选条件
4. **DataTable**：课程列表展示
5. **Dialog**：详情弹窗
6. **Card**：问答卡片

### 5.3 回答者显示逻辑

```dart
// 课前问答
if (qaType == QAType.preCourse) {
  if (question.answererType == AnswererType.customerService) {
    return '📝 客服';
  } else if (question.answererType == AnswererType.instructor) {
    return '👨‍🏫 ${question.answererName}';
  }
}

// 课后问答
if (qaType == QAType.postCourse) {
  // 课后问答只有讲师回答
  return '👨‍🏫 ${question.answererName}';
}
```

### 5.4 性能优化

1. 列表分页加载（每页10条）
2. 搜索防抖（500ms）
3. 图片懒加载
4. 筛选结果缓存

## 六、交互流程

### 6.1 课前问答流程

```
用户操作：
1. 进入课程问答管理页面
2. 默认显示"课前问答"Tab
3. 可选：使用搜索/筛选条件
4. 查看课程列表（展示问答统计）
5. 点击"详情"查看该课程的所有问题
6. 查看问题详情（回答者可能是客服或讲师）
7. 可选：回复待解决问题
```

### 6.2 课后问答流程

```
用户操作：
1. 切换到"课后问答"Tab
2. 可选：使用搜索/筛选条件
3. 查看课程列表（展示问答统计）
4. 点击"详情"查看该课程的所有问题
5. 查看问题详情（回答者都是讲师）
6. 点击"回复"回答学员问题
7. 标记问题为已解决
```
