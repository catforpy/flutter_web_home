## 🎯 V2接口特点

### 1. 明确区分不同身份

每种身份有独立的接口路径：
- 都达网账户：`/api/auth/v2/platform-account/*`
- 服务商：`/api/auth/v2/service-provider/*`
- 平台管理员：`/api/auth/v2/platform-admin/*`
- 后台管理员：`/api/auth/v2/backend-admin/*`

### 2. 明确区分登录方式

每种登录方式有独立的接口：
- 账号密码：`/login/password`
- 手机验证码：`/login/sms`

### 3. 独立的DTO（数据传输对象）

- `PasswordRegisterReqDTO` - 账号密码注册
- `PasswordLoginReqDTO` - 账号密码登录
- `SmsCodeLoginReqDTO` - 手机验证码登录

### 4. 通用接口共享

以下接口所有身份共享：
- 登出
- 刷新Token
- 验证Token
- 获取用户信息
- 发送验证码

## 💻 使用示例

### 都达网账户登录

```dart
final authState = AuthState();

// 方式1: 账号密码登录
final success = await authState.loginPlatformAccount(
  username: 'testuser',
  password: 'pass123456',
);

// 方式2: 手机验证码登录
final success = await authState.loginPlatformAccountWithSms(
  phone: '13900139000',
  verifyCode: '123456',
);

if (success) {
  print('登录成功！');
  print('用户名: ${authState.username}');
  print('用户类型: ${authState.getUserTypeName()}');
}
```

## 📊 与V1接口的区别

### V1接口（通用）

```
POST /api/auth/login
{
  "loginType": "account_password",
  "userType": "normal",
  "username": "testuser",
  "password": "pass123456"
}
```

**特点**：
- ✅ 一个接口支持所有用户类型和登录方式
- ❌ 需要手动指定loginType和userType
- ❌ 容易出错

### V2接口（明确区分）

```
POST /api/auth/v2/platform-account/login/password
{
  "username": "testuser",
  "password": "pass123456"
}
```

**特点**：
- ✅ 每种身份有独立接口
- ✅ 接口路径本身就说明了身份和登录方式
- ✅ 不需要额外参数
- ✅ 更清晰、更安全
