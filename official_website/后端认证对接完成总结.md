## 📦 Web端Token存储方案

### 技术实现

**使用的包**: `shared_preferences: ^2.3.4`

**存储方式**:
- **Web端**: 自动使用浏览器的 `localStorage`
- **移动端**: 使用原生的SharedPreferences

**存储的Token**:
- `auth_access_token` - 访问令牌
- `auth_refresh_token` - 刷新令牌
- `auth_token_expire_time` - Token过期时间戳

**存储的用户信息**:
- `user_info` - 完整的用户信息（JSON格式）

### 安全性说明

**当前实现**:
- ✅ Token存储在localStorage中（持久化）
- ✅ 刷新浏览器后Token仍然有效
- ⚠️  未加密存储（后续可优化）

**建议优化**（生产环境）:
1. 使用HttpOnly Cookie存储RefreshToken
2. 使用Short-lived Token（短期AccessToken）
3. 添加Token加密
4. 添加CSRF保护

## 🔄 认证流程说明

### 登录流程

```
1. 用户输入账号密码
   ↓
2. AuthState.login()
   ↓
3. AuthService.login()
   ↓
4. 后端API: POST /api/auth/login
   ↓
5. 后端返回: LoginResponseDTO
   - userId, username, realName, userType
   - accessToken, refreshToken, expiresIn
   ↓
6. TokenManager.saveTokens()
   - 存储到localStorage（Web端）
   - 更新内存缓存
   ↓
7. AuthState更新状态
   - _isLoggedIn = true
   - _userInfo = UserInfo
   - notifyListeners()
   ↓
8. UI自动更新，显示登录状态
```

### Token刷新流程

```
1. 任何API请求收到401错误
   ↓
2. ApiClient拦截器捕获401
   ↓
3. AuthService.refreshToken()
   ↓
4. 后端API: POST /api/auth/refresh
   - 参数: refreshToken
   ↓
5. 后端返回新的Token
   ↓
6. TokenManager.setAccessToken()
   - 更新localStorage
   - 更新内存缓存
   ↓
7. 重试原请求（使用新Token）
   ↓
8. 成功返回数据
```

## 📝 使用示例

### 示例1: 用户登录

```dart
// 在你的UI代码中
final authState = AuthState();

// 账号密码登录
final success = await authState.login(
  username: 'testuser',
  password: 'password123',
  userType: 'normal',
);

if (success) {
  print('登录成功: ${authState.username}');
  print('用户类型: ${authState.getUserTypeName()}');
} else {
  print('登录失败: ${authState.errorMessage}');
}
```

### 示例2: 手机验证码登录

```dart
// 发送验证码
final authService = AuthService();
final sendResult = await authService.sendSmsCode('13900139000');

// 验证码登录
final success = await authState.loginWithSms(
  phone: '13900139000',
  verifyCode: '123456',
  userType: 'normal',
);
```

### 示例3: 用户注册

```dart
// 注册新用户
final success = await authState.register(

  phone: '13900139000',

if (success) {
  print('注册成功，已自动登录');
}
```

